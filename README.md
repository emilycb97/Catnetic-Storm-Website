# Catnetic Storm - Official Website

Dynamiczna platforma webowa dedykowana grze typu roguelike "Catnetic Storm". Witryna łączy funkcjonalność informacyjną z rozbudowanym systemem społecznościowym, umożliwiając graczom interakcję, śledzenie postępów prac (devlog) oraz dzielenie się twórczością w galerii. Projekt został zbudowany w architekturze klient-serwer z wykorzystaniem bazy danych do zarządzania treścią w czasie rzeczywistym.

Projekt został zrealizowany w zespole trzyosobowym:
* **Emilia Szczerba** – [@emilycb97](https://github.com/emilycb97)
* **Klaudia Adamek** - [@klaudiaadamek](https://github.com/klaudiaadamek)
* **Hanna Sauchuk** 

## Funkcjonalności 
* **System użytkowników:** Pełna obsługa rejestracji i logowania z bezpiecznym przechowywaniem haseł oraz zarządzaniem sesjami.
* **Forum społecznościowe:** Interaktywny moduł dyskusyjny pozwalający na publikowanie, edytowanie i usuwanie postów (pełny system CRUD) wraz z obsługą załączników multimedialnych.
* **Devlog z systemem opinii:** Sekcja aktualizacji od twórców, gdzie zalogowani użytkownicy mogą zostawiać komentarze i wchodzić w interakcję z autorami gry.
* **Multimedialna galeria:** dynamiczny podgląd grafik i wideo z gry zaimplementowany za pomocą autorskiego skryptu Lightbox dla zapewnienia płynnych efektów wizualnych.
* **Zarządzanie bazą danych:** Strukturalne podejście do danych (SQL) zapewniające relacyjność między użytkownikami, postami na forum a komentarzami.

## Technologie 
* **Backend:** PHP (z wykorzystaniem PDO dla bezpiecznej komunikacji z bazą)
* **Frontend:** HTML5, CSS3, JavaScript (ES6)
* **Baza danych:** MySQL / MariaDB
* **Inne:** Zarządzanie sesjami użytkowników, obsługa plików (upload zdjęć i wideo)

*Data realizacji: Marzec 2026*

# Catnetic Storm - Official Website 

A dynamic web platform dedicated to the "Catnetic Storm" roguelike game. The website integrates informative content with an extensive social system, allowing players to interact, follow development progress via devlogs, and share their creations in a media gallery. The project was built using a client-server architecture with a database for real-time content management.

Project developed in a three-person team:
* **Emilia Szczerba** – [@emilycb97](https://github.com/emilycb97)
* **Klaudia Adamek** - [@klaudiaadamek](https://github.com/klaudiaadamek)
* **Hanna Sauchuk** 

## Features 
* **User Authentication:** Full registration and login system with secure password handling and session management.
* **Community Forum:** An interactive discussion module allowing users to create, read, update, and delete posts (full CRUD) with multimedia attachment support.
* **Devlog with Feedback:** A dedicated section for game updates where logged-in users can leave comments and interact with the developers.
* **Multimedia Gallery:** Dynamic preview of game art and videos implemented using a custom Lightbox script for a smooth visual experience.
* **Database Management:** Structured data approach (SQL) ensuring relational integrity between users, forum posts, and comments.

## Technologies 
* **Backend:** PHP (using PDO for secure database communication)
* **Frontend:** HTML5, CSS3, JavaScript (ES6)
* **Database:** MySQL / MariaDB
* **Other:** User session management, file handling (image and video uploads)

*Developed: March 2026*
